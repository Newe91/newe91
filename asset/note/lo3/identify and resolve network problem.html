Theory

What is computer network?

Computer network is a group of computers connected together.
The purpose of computer network

  • To sharing resource such as printer
  • File sharing
  • Increased storage capacity and etc...

What is network monitoring?

Network monitoring refers to the practice of overseeing the operation of a computer network using specialized management software tools. Network monitoring allows network managers to collect and analyze network activity statistics in order to be better informed so that they can make sound management decision.
Network security monitors or detectors monitor and alert of any suspicious or malicious activity on the network such as:- Intrusions,Attacks and breaches.

Recommendations for improving network performance

  • Eliminate performance bottlenecks
  • Reconfigure your network hardware
  • Communicate with your users on proper network usage
  • Upgrade network device
  • Provide a separate guest network and etc...

Ways to measure network performance

  • Network availability testing network availability testing may be as simple as running a ping test that requests responses from your network devices to ensure that everything is connected and operational.
  • Network speed test will check the network upload and download speeds.
  • Bandwidth usage testing by configuring the network you may have identified how much bandwidth can be used at once. Bandwidth optimization tools: SolarWinds tools,solarWinds engineers toolset, paessler PRTG manageEngine tools and Ipswitch WhatsUp Gold.

Network monitoring software and devices categorized

  • Network monitoring platforms are complete solutions that can integrate configuration management, monitoring change management and etc...
  • Network monitoring utilities generally include specialist software that can perform specific tasks such as protocol analysis, connectivity tools(ping,tracert), log analysis and reporting and etc..

Network monitoring system and tools

  • Diagnostic tools used to test connectivity, determine that a location is reachable or a device is up usually active tools.
    Network diagnostic tools
    • Ping used to test the reachability of a host on an Internet Protocol(IP) network.
    • Ipconfig short for Internet Protocol Configuration used in Windows operating systems to display and manage the network configuration of a device. It provides information about the device's IP address, subnet mask, default gateway and DNS servers.
    • Tracert short for traceroute is a network diagnostic tool used to track the pathway taken by data packets from your computer to a destination server or website. It is commonly used in troubleshooting network connectivity issues, identifying network latency problems and determining the route that data packets take across the internet.
    • Netstat Netstat can display active network connections, including the local and remote addresses and the state of the connection.
    • Nmap network mapper used to discover devices and services on a computer network creating a map of the network.
    • Whois is a protocol used to query databases that store information about registered domain names, IP addresses and autonomous system numbers. The primary purpose of Whois is to look up information related to domain names.
    • Nslookup which stands for Name Server Lookup. It is commonly used to troubleshoot DNS-related issues, verify DNS records and gather information about domain names and their associated IP addresses.
    • Tracerouter is a network diagnostic tool that is used to determine the path that a packet takes from the source to the destination across an IP network.
    • Nbtstat used to diagnose and troubleshoot NetBIOS over TCP/IP issues. Nbtstat stands for NetBIOS over TCP/IP Statistics and it can display various statistics related to the NetBIOS over TCP/IP functionality of the system.
    • Pathping it combines the functionality of the tracert and ping commands to provide detailed information about network latency and packet loss along the route to a specified destination.
    • Tcpdump a utility that is used to obtain packet information from a query string sent to the network interface.
  • Monitoring tools running in the background services which collect events, but can also initiate their own searches by using diagnostic tools and recording output in a scheduled.
      Monitoring tools include:-
    • Network analyzers or sniffers
    • Network mappers or scanners
    • Network performance monitors or testers

Network troubleshooting applications

  • Packet sniffer provides a comprehensive view of a given network. You can use this application to analyze traffic on the network.
  • Port scanner looks for open ports on the target device and gathers information, including whether the port is open or closed.
  • Protocol analyzer use analyzers to troubleshoot network problems and detect intrusions into your network.
  • Wi-Fi analyzer detects devices and points of interference in a Wi-Fi signal. This tool can help you to troubleshoot issues in network connectivity over a wireless network.
  • Bandwidth speed tester tests the bandwidth and latency of a user internet connection. This application is typically accessed through a third-party website and can be used to confirm user reports about slow connections or download speeds.

The most common network problems

  • Hardware problems Cable is faulty or improperly connected.
  • Configuration problem misconfiguration of hardware and software in network.
  • Excessive network collisions these often lead to slow connectivity. The problem can occur as a result of bad network setup/plan, a user transferring a lot of information or jabbering network card.
  • Software problem can often be traced to software configuration such as DNS configuration, WINS configuration, the registry etc.

Troubleshooting methodology

  • Proceed logically and methodically
  • Follow recommended steps
  • Use experience when necessary
  • Logical approach benefits
  • Prevents wasteful, time-consuming efforts

Network troubleshooting process

  • Define the problem
  • Documenting the history of the problem
  • Analyzing the current environment
  • Correcting the problem
  • Testing the corrective action
  • Following up
  • Document the problem and the solution

Approaches to network fault finding

Cyclic fault finding methods are the most widely approved methods for fault finding across many technical area, not just ICT industries.

The scientific method to fault finding

Fault finding is a skill required in just about every industry, not just in ICT support. In general those areas of industry which demand high level technical skills have developed well documented fault finding methods.

The Scientific method proposes to use logical and systematic steps, to analyse available information. This procedure will enable you to draw conclusions and hopefully arrive at the source of the problem. Generally the method is repeated until the source of the problem has been indentifed.

Principles of the scientific method

  • Gather information
  • State the problem
  • Form a hypothesis
  • Test the hypothesis
  • Draw conclusions
  • Repeat when necessary

Why network maintenance need?

  • To prevent problem
  • Work efficiently
  • Maintain security
  • Stay up to date

The six important areas of network management

  • Software management this involves taking care of the software. It includes keeping track of installed software, applying any important updates and deciding whether to upgrade when new versions become available.
  • Hardware management it's involves inspecting servers, cleaning dust from vents and testing key hardware like uninterruptible power supplies and backup drivers.
  • Device management
  • File management involves keeping all the files on your system organized and deleting temporary files.
  • Security management task include running and testing backups, regularly scanning for viruses and testing your firewall.
  • User management task include reduce administration boost your network's security further by giving employees different levels of access depending on what they need to do, removing access rights when staff leave and controlling what files people can change.
