Unit 1: Introduction to computer network

Definition of computer network

Network is a group of connected computers that allows people to share resources.

Advantage of a computer network

  • Sharing hardware such as printer
  • File sharing
  • Increased storage capacity
  • Centralized administration and support
  • Increased cost efficiency and etc...

Disadvantage of a computer network

  • Security issue
  • Rapid spread of computer virus
  • Expensive setup
  • Dependency on the main servers and etc...

Server and client and others terms

Server provide shared resources to network users.
Clients access shared network resources provided by a server.
Resources is any service or device, such as files, printers, or other items, made available for use by members of the network.
Media is wires that make the physical connections.

Type of network based on the area coverage of the network

  • Local Area Network(LAN) used to interconnect distributed communities of computers located within a single building.
  • Metropolitan area Network(MAN) a larger network that usually spans several buildings in the same city.
  • Wide area Network(WAN) it's cover larger area best example is internet.

Types of network based on configuration

  • Peer-to-Peer network is also called Workgroup. There are no dedicated servers and there is no hierarchy among the computers. All the computers are equal and therefore are known as peers.
    Advantages of peer to peer network
    • Easy to install and configure
    • The cost of installation and operation is less
    • A full time network administrator is not required
    Disadvantages of peer to peer network
    • Shared resources can be accessed by everyone
    • Backup has to be performed on each computer separately
    • No centralized security
  • Client/server network is also called Domain. There are dedicated servers and there is hierarchy among the computers.
    Advantages of server based network
    • Centralized resources
    • Efficient
    • Security
    • Scalability
    Disadvantages of server based network
    • If the server goes down, it takes part or the whole network with it
    • It is more expensive to install
    • Needs to be maintained by staff with high IT skills

Network topology

Network topology refers to the arrangement of computers, cables and other components on the network.

Category of topology

  • Logical: describes the way network data flows through the physical components.
  • Physical: describes how the physical components on a network are connected.

Common types of topology

  • Bus topology is often referred to as a linear bus because the computers are connected in a straight line.
    Advantage of bus topology
    • A bus network topology is the cheapest of all topologies
    • No special configuration is required
    • It is easy to install and no special equipment is needed for installation
    • It needs less cable length than do other topologies
    Disadvantage of bus topology
    • A break in cable or a missing terminator can bring down the entire network
    • It is not possible to add or remove computers without disrupting the network
    • It is difficult to troubleshoot and administer
    • Adding of more computers degrades performance
  • Ring topology connects computers on a single circle of cable.
    Advantage of ring topology
    • Fast Execution
    • Better administration
    • Easy to install
    Disadvantage of ring topology
    • Quite expensive
    • Slow activity rate
    • Unprotected use
    • Difficult to troubleshoot
    • Break in a cable or fault computer can bring slow down entire network
  • Star topology computer are connected to a centralized component called a hub or switch.
    Advantage of star topology
    • Easy to install
    • No disruptions to the network when adding and removing devices
    • Easy to detect faults and to remove parts
    Disadvantage of star topology
    • If the centralized point fails the network fails
    • Requires more cable length than a linear topology
    • More expensive than linear bus topologies because of the cost of the hubs
  • Mesh topology offers superior redundancy and reliability. In a mesh topology, each computer is connected to every other computer by separate cabling. Advantage of mesh topology
    • Redundancy and reliability
    • A mesh network is highly reliable because of redundant multiple paths between computers
    • The failure of a single computer or a cable fault does not affect network operations
    • Computers can be added or removed without affecting the network
    Disadvantage of mesh topology
    • Uses a lot of cabling
    • It is difficult to install and troubleshoot
    • It is very expensive because of the length of cable required to make multiple redundant connections
    • Only a limited number of computers can be connected in a mesh topology
  • Hybrid topology it is a combination of two or more standard topologies for example star bus.
    Advantages of hybrid topology
    • Reliable to use
    • Any topology can be combined to make a new network
    • Scalable and very effective
    Disadvantages of hybrid topology
    • Costly very expensive
    • Difficult to install
    • Occur cable failure

Network transmission media

  • Guided transmission media data transmission is through wired system.
  • Unguided transmission media data transmission through wireless system.
    Types of wireless system
    • Infrared transmission
    • Laser transmission

Commonly types of cables

  • Twisted pair surrounded by insulation, a braided metal shielding and an outer cover.
    There are two types of coaxial cable:-
    • Thin(thinnet) cable
    • Thick(thicknet) cable
  • Coaxial cable consists of two insulated strands of copper wire twisted around each other.
    Two types of twisted pair cable:-
    • Shielded twisted pair cable(STP)
    • Unshielded twisted pair cable(UTP)
      There are five categories of unshielded twisted pair cable(UTP):-
      • Category 1(Cat1) This refers to traditional UTP telephone cable that can carry voice but not data transmissions. Most telephone cable prior to 1983 was Category 1 cable.
      • Category 2(Cat2) This category certifies UTP cable for data transmissions up to 4 megabits per second (Mbps). It consists of four twisted pairs of copper wire.
      • Category 3(Cat3) This category certifies UTP cable for data transmissions up to 16 Mbps. It consists of four twisted pairs of copper wire with three twists per foot.
      • Category 4(Cat4) This category certifies UTP cable for data transmissions up to 20 Mbps. It consists of four twisted pairs of copper wire.
      • Category 5(Cat5) This category certifies UTP cable for data transmissions up to 100 Mbps. It consists of four twisted pairs of copper wire.
  • Optical fiber optical fibers carry digital data signals in the form of modulated pulses of light. It was high speed cable.

Consideration for cable selection

  • Installation logistics
  • Shielding
  • Crosstalk
  • Transmission rates
  • Cost and etc...

Unit 2: Determine user needs

Mode of transmission

The way in which data is transmitted from one place to another is called data transmission mode.

Types of mode of transmission

  • Simplex mode data is sent in one direction only from sender to receiver. Senders cannot even be sure that the data is received. Examples of simplex transmission are radios, walkie-talkies and television.
  • Half-Duplex mode the communication can take place in both directions but only in one direction at a time. Example internet browser.
  • Full-Duplex mode the communication can take place in both directions simultaneously. Example telephone.

Parallel vs serial transmission

  • Parallel transmission bits of data flow concurrently through separate communication lines. parallel transmission is commonly used to transfer data from computer to printer.
  • Serial transmission bits of data flow in sequential order through single communication line. Serial transmission is typically slower than parallel transmission because data is sent sequentially in a bit-by-bit fashion. Serial mouse uses serial transmission mode in computer.

Data transfer methods

  • Circuit switching it is not broken up.
    Advantages of circuit switching
    • Circuit is dedicated to the call no interference no sharing
    • Guaranteed the full bandwidth for the duration of the call
    • Guaranteed quality of service
    Disadvantages of circuit switching
    • The equipment may be unused for a lot of the call, if no data is being sent, the dedicated line still remains open
    • Takes a relatively long time to set up the circuit
    • During a disaster the network may become unavailable.
  • Packet switching the message gets broken into small data packets.
    Advantages of packet switching
    • Security
    • Bandwidth used to full potential
    • Devices of different speeds can communicate
    • Not affected by line failure
    • Do not have to wait for a direct connection to become available
    • During a disaster, when the public telephone network might stop working, emails and texts can still be sent via packet switching
    Disadvantages of packet switching
    • Under heavy use there can be a delay
    • Data packets can get lost or become corrupted
    • Protocols are needed for a reliable transfer
    • Not so good for some types data streams e.g real-time video streams can lose frames due to the way packets arrive out of sequence.

Synchronous and asynchronous data transmission

  • Synchronous data transmission the sender and receiver operate on the same clock signal. This means that both devices share the same time reference, which allows them to transmit and receive data simultaneously.
  • Asynchronous data transmission the sender and receiver do not share a common clock signal. Instead, data is transmitted in separate packets or frames, with each packet containing start and stop bits that indicate the beginning and end of the data.

The open system interconnection(OSI) reference model

The OSI reference model represents the seven layers of the process by which data is packaged and transmitted from a sending application through the physical wires to the receiving application.

The seven layers of the OSI reference model

  • Application layer this layer relates to the services that directly support user applications such as software for internet explorer and email.
  • Data link layer makes to application to open, use and close a connection.
  • Network layer responsible for the delivery of packets from the source host to the destination host based on their IP addresses.
  • Transport layer responsible for end-to-end communication between applications running on different devices.
  • Session layer responsible for establishing, maintaining and terminating connections between two devices.
  • Presentation layer responsible for data formatting, encrypting and compressing data before it is sent across the network.
  • Physical layer responsible for transmitting raw data bits over a physical medium.

Protocol

Protocol a set of rule and procedures which govern communication between computers.

Common types of protocols

  • User Datagram Protocol(UDP) connectionless protocol
  • Transmission control protocol(TCP) connection based protocol
  • Internet Protocol(IP) is a packet-switched protocol that performs addressing and route selection.
  • IP(Internet Protocol) an address is a unique identifier assigned to each device connected to a network that uses the Internet Protocol for communication.
  • Simple Mail Transfer Protocol(SMTP) used to send and receive e-mail.
  • Post Office Protocol(POP3) 3 stand for version 3 ,retrieves emails from a server to a client device.
  • Internet Message Access Protocol(IMAP) is a protocol used for accessing email messages on a mail server by an email client.
  • File Transfer Protocol(FTP) is a protocol used for transferring files between computers on a network.
  • Simple Network Management Protocol(SNMP) used for network management.
  • Hypertext Transfer Protocol(HTTP) is the protocol used for transmitting hypertext requests and information between servers and browsers.
  • Hypertext Transfer Protocol Secure(HTTPS) is a secure version of HTTP.
  • Internet Control Message Protocol(ICMP) used by IP and higher-level protocols to send and receive status reports about information being transmitted.
  • Domain Name System(DNS) translates domain names into IP addresses to locate resources on the internet.
  • Dynamic Host Configuration Protocol(DHCP) assigns IP addresses dynamically to devices on a network.
  • Address Resolution Protocol(ARP) determines hardware address(MAC addresses) that correspond to an IP address
  • Reverse Address Resolution Protocol(RARP) provides an IP number to a requesting hardware address.
  • Telnet is a network protocol that allows users to connect to remote computers and access their resources over a network and etc...

Transmission control protocol/internet protocol(TCP/IP)

The TCP/IP protocol does not exactly match the OSI reference model. Instead of seven layers, it uses only four.

  • Network interface layer corresponding to the physical and data-link layers of the OSI reference model.
  • Internet layer corresponding to the network layer of the OSI reference mode.
  • Transport layer corresponding to the transport layer of the OSI reference mode.
  • Application layer Corresponding to the session, presentation and application layers of the OSI reference model.

Unit 3: Develop best topology

Connectivity devices

Network interface cards(NIC)

Also called network adapter receive electrical signals and convert them into data. Network interface cards(NIC) connectivity device is mandatory in network.

Switch

A switch is a network device that connects devices together on a computer network by using packet switching to receive, process and forward data to the destination device.

Hub

Hub is used as the central hardware component in a star topology.

Repeater

Repeater use for extend the length to which the signal can be transmitted over the same network.

Bridges

Bridge in computer network is a smart device that is used for creating one bigger network by connecting smaller subnetworks.

Router

Router is a device that connect two or more packet switch network or subnetwork.

Gateway

Gateway enable communication between different architectures and environments.

Media Access Control(MAC) address

MAC addresses are the physical addresses of network devices.

IP address

An IP address consists of two parts a network address that identifies the network and a host address that identifies.

Types of IP address

  • Internet protocol version 4(IPV4) source and destination addresses are 32 bits (4 bytes) in length.
  • Internet protocol Version 6(IPV6) source and destination addresses are 128 bits (16 bytes) in length.

Network classes

  • Class A network
    • Have an 8-bit network ID and a 24-bit host ID
    • Allowing for a large number of hosts over 16 million on each network
    • This class will only support 126 networks and each network can support 16,777,214 hosts
    • The possible ranges 1.0.0.0 - 127.255.255.255
    • The default subnet mask for Class A networks is 255.0.0.0
  • Class B network
    • Class B networks have a 16-bit network ID and a 16-bit host ID
    • Allowing for a moderate number of hosts over 65,000 on each network
    • This class will support 16,384 networks and limited to 65,534 hosts per network
    • The possible ranges 128.0.0.0 - 191.255.255.255
    • The default subnet mask for Class B networks is 255.255.0.0
  • Class C network
    • Class C networks have a 24-bit network ID and an 8-bit host ID
    • Allowing for a small number of hosts up to 254 on each network
    • The possible ranges 192.0.0.0 - 223.255.255.255
    • The default subnet mask for Class C networks is 255.255.255.0
  • Class D network
    • Class D networks are used for multicasting which is a method of sending data to multiple destinations simultaneously
    • There can be up to 268 million multicast groups in a Class D network
    • The possible ranges 224.0.0.0 - 255.255.255.254
  • Class E network
    • Class E networks are reserved for future use and are not currently in use
    • The possible ranges 224.0.0.0 - 255.255.255.254

Public and private IP address

  • Private IP address are typically used on local networks including home, school and business LANs. Private networks are non routable. Devices with private IP addresses cannot connect directly to the Internet.
  • Public IP address for use on the Internet or other wide area network (WAN).

Static and Dynamic IP address

  • Static IP address
    • A static IP address is a permanent unchanging IP address assigned to a specific device.
    • This type of IP address is manually configured and does not change unless it is manually changed by the user or network administrator.
  • Dynamic IP address
    • A dynamic IP address is a temporary IP address assigned by a Dynamic Host Configuration Protocol (DHCP) server
    • This type of IP address changes each time the device connects to the network or after a certain period of time.

Subnet mask

A subnet mask or sub network mask is a 32 bit number which is used to identify which portion of the IP address identifies the network portion and which part indicates the host part or portion.

Subnneting

The process of breaking down an IP network into smaller sub-networks called subnets. Subnetting a technique that allows a network administrator to divide one physical network into smaller logical network.
